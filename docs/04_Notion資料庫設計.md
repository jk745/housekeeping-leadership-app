# Notion 資料庫設計

建議建立一個主頁：

# 房務部轉任準備中心

裡面放 5 個資料庫：

1. 30 天轉任進度表
2. 《如何領導》讀書筆記
3. 房務現場觀察紀錄
4. 團隊信任建立紀錄
5. 改善議題追蹤表

---

## 1. 30 天轉任進度表

### 欄位建議

| 欄位 | 類型 | 說明 |
|---|---|---|
| 任務名稱 | Title | 例如：跟房務主管訪談一次 |
| 週次 | Select | 第1週、第2週、第3週、第4週 |
| 分類 | Select | 讀書、觀察、訪談、實作、整理、回報 |
| IPA | Select | 構想 Idea、人才 People、行動 Action |
| 狀態 | Status | 未開始、進行中、已完成 |
| 優先度 | Select | 高、中、低 |
| 日期 | Date | 預計完成日 |
| 對應讀書筆記 | Relation | 連到《如何領導》讀書筆記 |
| 對應現場觀察 | Relation | 連到房務現場觀察 |
| 對應改善議題 | Relation | 連到改善議題追蹤表 |
| 我的反思 | Text | 完成後寫下學到什麼 |

### 建議視圖

| 視圖名稱 | 類型 | 用途 |
|---|---|---|
| 本週行動 | Board / Table | 只看這週要做什麼 |
| 依週次查看 | Board | 第1週～第4週分欄 |
| 依 IPA 查看 | Board | 構想、人才、行動分欄 |
| 已完成回顧 | Table | 看自己累積了什麼 |

## 2. 《如何領導》讀書筆記

### 欄位建議

| 欄位 | 類型 | 說明 |
|---|---|---|
| 筆記標題 | Title | 例如：領導不是職位，而是行為 |
| 章節 | Text | 書中章節或頁碼 |
| IPA | Select | 構想 Idea、人才 People、行動 Action |
| 核心觀念 | Text | 這段在講什麼 |

| 可執行行動 | Text | 下週可以做什麼 |
| 連結任務 | Relation | 連到 30 天進度表 |
| 重要程度 | Select | 高、中、低 |

## 3. 房務現場觀察紀錄

### 欄位建議

| 欄位 | 類型 | 說明 |
|---|---|---|
| 觀察主題 | Title | 例如：退房清潔流程 |
| 日期 | Date | 觀察日期 |
| 場景 | Select | 退房、續住、查房、備品、布巾、遺留物、維修、房況 |
| 我看到的流程 | Text | 實際怎麼做 |
| 我不懂的地方 | Text | 需要請教什麼 |
| 對客務的影響 | Text | 這件事如何影響櫃檯與客人 |
| 可改善處 | Text | 有沒有可以調整的地方 |
| 連結任務 | Relation | 連到 30 天進度表 |
| 連結改善議題 | Relation | 連到改善議題追蹤表 |

### 建議視圖

| 視圖名稱 | 用途 |
|---|---|
| 依場景分類 | 看退房、查房、備品、維修等 |
| 我還不懂的事 | 專門整理需要請教的問題 |
| 可改善處 | 轉成改善議題 |

## 4. 團隊信任建立紀錄

### 欄位建議

| 欄位 | 類型 | 說明 |
|---|---|---|

## 5. 改善議題追蹤表

前 30 天以觀察為主，先不要急著改革。

### 欄位建議

| 欄位 | 類型 | 說明 |
|---|---|---|
| 議題名稱 | Title | 例如：急房回報不夠即時 |
| 類型 | Select | 房況、清潔、查房、備品、布巾、遺留物、維修、跨部門溝通 |
| 影響對象 | Multi-select | 客人、櫃檯、房務、工程、主管 |
| 嚴重度 | Select | 高、中、低 |
| 發生頻率 | Select | 經常、偶爾、少見 |
| 目前狀況 | Text | 現在怎麼處理 |
| 可能原因 | Text | 為什麼會發生 |
| 初步想法 | Text | 可以怎麼改善 |
| 狀態 | Status | 觀察中、待討論、可改善、已完成 |
| 連結觀察 | Relation | 連到房務現場觀察 |
| 連結任務 | Relation | 連到 30 天進度表 |

## Notion 首頁建議文字

```markdown
# 房務部轉任準備中心

> 我的目標，不是成為最會做房的人，而是成為最能理解現場、整合團隊、穩定品質的人。

## 本週重點
連結：30 天轉任進度表｜篩選本週

## 今日小進度
- 讀書 10 分鐘
- 寫 3 行筆記
- 記 1 個現場觀察
- 問 1 個實務問題
- 想 1 個可以改善的小地方

## IPA 領導框架
- 構想 Idea：我要把房務部帶成什麼樣子？
- 人才 People：我要如何建立信任？
- 行動 Action：我下一步要做什麼？

## 資料庫
- 30 天轉任進度表
- 《如何領導》讀書筆記
- 房務現場觀察紀錄
- 團隊信任建立紀錄
- 改善議題追蹤表

## 每週檢核
- 我這週學到什麼？
- 我還不懂什麼？
- 團隊最在意的是什麼？
- 我下週最重要的一步是什麼？
```
