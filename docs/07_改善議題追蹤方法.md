# 改善議題追蹤方法

前 30 天不要急著改革。先把問題看清楚，再討論改善。

## 改善議題不是抱怨

請把問題整理成這個格式：

1. 發生什麼事？
2. 可能原因是什麼？
3. 影響誰？
4. 發生頻率高不高？
5. 嚴重度如何？
6. 現在如何處理？
7. 有沒有小幅改善的方法？
8. 需要跟誰討論？

## 優先改善判斷

| 判斷面向 | 問題 |
|---|---|
| 影響客人體驗 | 會不會造成客訴或入住不順？ |
| 影響房況速度 | 會不會造成可售房延遲？ |
| 影響同仁壓力 | 會不會讓現場同仁一直補救？ |
| 發生頻率 | 是經常發生，還是偶爾發生？ |
| 改善難度 | 是小調整就能改善，還是需要跨部門決策？ |

## 不要一開始就做大改革

先找：

- 小問題
- 高頻率
- 低阻力
- 能減少同仁負擔
- 能改善客人感受
- 能讓客務與房務更順的議題

## 常見改善議題分類

- 房況回報
- 急房處理
- 清潔瑕疵
- 查房標準
- 備品補充
- 布巾不足
- 遺留物交接
- 維修通報
- 延退銜接
- 客務與房務溝通

## 改善議題紀錄格式

| 欄位 | 說明 |
|---|---|
| 議題名稱 | 這個問題叫什麼 |
| 類型 | 房況、清潔、備品、維修等 |
| 影響對象 | 客人、櫃檯、房務、工程、主管 |
| 嚴重度 | 高、中、低 |
| 發生頻率 | 經常、偶爾、少見 |
| 目前狀況 | 現在怎麼處理 |
| 可能原因 | 為什麼會發生 |
| 初步想法 | 可以怎麼改善 |
| 狀態 | 觀察中、待討論、可改善、已完成 |

## 改善原則

> 先整理問題，不急著定罪；先找小改善，不急著大改革。
